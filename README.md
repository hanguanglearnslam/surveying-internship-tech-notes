# Surveying Internship Tech Notes

A geospatial engineering portfolio documenting surveying data workflows, quality-check scripts, GIS platform testing,and 3D/point cloud data pipelines.

## Overview
This repository records my technical learning, workflow summaries, and reusable tools from a surveying and geospatial engineering internship.

The goal is to connect real-world surveying practice with future research directions in photogrammetry, 3D reconstruction, point cloud processing, GIS platforms, and geospatial AI.

This repository does **not** contain confidential project data, internal documents, client information, or non-publicscreenshots. All notes, scripts, and examples are written for learning, reproducibility, and technical documentation purposes.

  ## Goals

  - Understand real-world surveying and geospatial data workflows.
  - Document common data formats, processing steps, and quality-control issues.
  - Build small automation tools for repetitive technical support tasks.
  - Learn how GIS platforms and 3D geospatial systems are tested and deployed.
  - Connect engineering practice with future research in photogrammetry, 3D reconstruction, point clouds, SLAM, remote
  sensing, and geospatial AI.

## Main Topics

### 1. Surveying Data Workflows
This section documents typical geospatial data workflows, including:

  - Data collection
  - Data organization
  - Coordinate system checking
  - Format conversion
  - Quality inspection
  - Data delivery
  - Platform integration

  Possible data types include:

  - DOM / DSM / DEM
  - GeoTIFF
  - SHP / GeoPackage
  - LAS / LAZ point clouds
  - OSGB / 3D Tiles
  - UAV images
  - Remote sensing images
  - 3D reconstruction outputs

  ### 2. GIS Platform Testing

  This section focuses on how geospatial data is uploaded, published, visualized, and tested in GIS or 3D geospatial
  platforms.

  Typical testing topics include:

  - Data upload
  - Map service publishing
  - 3D model loading
  - Layer display
  - Attribute query
  - Coordinate alignment
  - Permission control
  - Bug reproduction
  - Platform performance issues

  ### 3. Point Cloud and 3D Data Pipelines

  This section records learning notes related to point cloud and 3D data processing.

  Potential topics include:

  - Point cloud format conversion
  - Noise checking
  - Coordinate alignment
  - Model simplification
  - 3D Tiles generation
  - OSGB model inspection
  - Web-based 3D visualization
  - Cesium / WebGIS platform integration

  ### 4. Automation Tools

  The scripts/ folder contains small tools for repetitive geospatial engineering tasks.

  Planned tools include:

  - File inventory generator
  - Folder structure checker
  - Delivery checklist generator
  - Batch file renaming tool
  - Data completeness checker
  - Log summary tool

  These tools are designed to improve efficiency in technical support and data management workflows.
